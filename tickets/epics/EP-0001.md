# [Epic] MPM Commander Phase 3: Interactive Chat Interface

## Description
## Overview
Phase 3 adds the interactive chat interface layer on top of the Phase 2 REST API infrastructure.

## User Expectations
When running `claude-mpm --commander`:
- a) Load user-level skills and agents
- b) Start context-managed, session-based chat interface using OpenRouter
- c) User can: list instances, connect to instance, send commands
- d) Start/connect to instance with project path + framework (cc, mpm, codex)
- e) Tmux session starts with selected framework loaded
- f) User receives LLM-summarized output from instances
- g) Commander acts as intelligent proxy between user and Claude Code instances

## Architecture
```
User - CommanderChatCLI (New) - REST API (Phase 2) - tmux - Claude Code Instances
```

## Success Criteria
- User can run `claude-mpm --commander` and get interactive prompt
- Can list, start, connect to instances
- Commands forwarded to Claude Code instances via tmux
- Output summarized via LLM before display

## Sub-Tickets
1. Implement OpenRouter LLM Client
2. Implement Chat Interface CLI
3. Implement Framework Selection & Instance Startup
4. Implement Output Proxy & Summarization
5. Implement Session Context Manager
6. Phase 3 Integration Testing & Documentation
