# feat: /mpm-integrate - Community-Driven API Integration Framework

## Description
## Vision
Enable users to declaratively create, share, and use REST API and GraphQL integrations without writing code. A "Zapier meets MCP" approach where YAML manifests define how to talk to any API.

## Problem Space
1. No unified integration discovery/browsing for new APIs
2. Integration templates are hardcoded in Python (MCPServiceRegistry)
3. No credential setup wizard (manual .env editing required)
4. No integration health monitoring
5. No community contribution path for integrations (unlike skills/agents)
6. Users can't add ad-hoc API integrations without code changes

## Success Metrics
- 10+ integrations in catalog within 3 months
- Community contributions (PRs for new integrations)
- Reduction in "how do I connect to X" support requests
- Integration health check pass rate > 95%

## Scope (Included)

### Project vs User Level Integrations
Integrations can be created at two levels:
- **User-level** (~/.claude-mpm/integrations/) - Available across all projects
- **Project-level** (.claude/integrations/) - Project-specific, can override user-level

Priority chain: Project-level > User-level > Catalog defaults

### Agent & Skill Pairing
Each integration automatically generates:
- **Agent**: {integration}-agent.md - Specialized agent for working with the integration
- **Skill**: {integration}.md - Skill with prompts for common operations

This creates a complete "integration bundle":
linear/
- integration.yaml      # API definition
- linear-agent.md       # Generated agent
- linear-skill.md       # Generated skill
- scripts/              # Batch operations

### Phase 1: Core Framework (MVP)
- Integration manifest parser (YAML format)
- Credential management with .env/.env.local wizard
- REST and GraphQL client implementation
- Basic CLI: add, remove, call, status, list
- Local integration storage in ~/.claude-mpm/integrations/
- Catalog structure in src/claude_mpm/integrations/catalog/
- **Agent & Skill Generator** - Auto-generate paired agent and skill files for each integration

### Phase 2: MCP Generation
- Auto-generate MCP server wrappers from manifests
- Register generated servers with .mcp.json
- Tool discovery for Claude to use integrations directly

### Phase 3: Community & Batch
- Contribution workflow (TEMPLATE.yaml, CONTRIBUTING.md)
- Auto-generated catalog index (_index.yaml)
- Batch script framework for bulk operations
- create wizard for new integrations
- validate command for PR validation
- CI integration for catalog validation

### Phase 4: Advanced (Future)
- OAuth2 flow support
- Rate limiting / retry logic
- Response caching
- Integration bundles/profiles

## Out of Scope
- Migration of existing setup commands (notion, slack, gworkspace remain unchanged)
- Encrypted credential store (use .env pattern)
- System keychain integration
- Separate integrations repository (keeping in main repo)

## Technical Architecture

### Directory Structure
src/claude_mpm/
- integrations/
  - core/
    - manifest.py          # Integration manifest parser
    - client.py            # REST/GraphQL client
    - credentials.py       # .env management
    - mcp_generator.py     # Auto-generate MCP wrappers
    - generator.py         # Agent & Skill generator
  - catalog/                 # Community contributions
    - README.md
    - TEMPLATE.yaml
    - linear/
    - airtable/
    - _index.yaml          # Auto-generated
  - cli/
    - integrate.py

## Dependencies
- Existing .env loading infrastructure
- MCP server generation patterns (reference: existing MCP services)
- Skill-like deployment pattern (cache -> deploy)

## Risks & Mitigations
| Risk | Mitigation |
|------|------------|
| API changes break integrations | Version field + health checks |
| Credential security | .env.local (gitignored), no encryption needed |
| Contribution quality | Validation in CI, review guidelines |

- Phase 2: 1-2 weeks
- Phase 3: 1-2 weeks
- Phase 4: Future backlog
