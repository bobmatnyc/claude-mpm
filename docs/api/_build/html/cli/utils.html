CLI Utils

CLI utilities provide common functionality used across all CLI commands.

Utils

Utility functions for the CLI.

WHY: This module contains shared utility functions used across different CLI commands. Centralizing these functions reduces code duplication and provides a single place for common CLI operations.

claude_mpm.cli.utils.get_user_input(input_arg: str | None, logger) → str[source]

Get user input based on command line arguments.

WHY: This function handles the three ways users can provide input: 1. Direct text via -i/–input 2. File path via -i/–input 3. stdin (for piping)

DESIGN DECISION: We check if the input is a file path first, then fall back to treating it as direct text. This allows maximum flexibility.

Parameters:

  • input_arg – The value of the -i/–input argument

  • logger – Logger instance for output

Returns:

The user input as a string

claude_mpm.cli.utils.get_agent_versions_display() → str | None[source]

Get formatted agent versions display as a string.

WHY: This function provides a single source of truth for agent version information that can be displayed both at startup and on-demand via the /mpm agents command. This ensures consistency in how agent versions are presented to users.

Returns:

Formatted string containing agent version information, or None if failed

claude_mpm.cli.utils.list_agent_versions_at_startup() → None[source]

List deployed agent versions at startup.

WHY: Users want to see what agents are available when they start a session. This provides immediate feedback about the deployed agent environment.

claude_mpm.cli.utils.setup_logging(args) → object[source]

Set up logging based on parsed arguments.

WHY: This centralizes logging setup logic, handling the deprecated –debug flag and the new –logging argument consistently across all commands.

Parameters:

args – Parsed command line arguments

Returns:

Logger instance

claude_mpm.cli.utils.ensure_directories() → None[source]

Ensure required directories exist on first run.

WHY: Claude-mpm needs certain directories to function properly. Rather than failing when they don’t exist, we create them automatically for a better user experience.
