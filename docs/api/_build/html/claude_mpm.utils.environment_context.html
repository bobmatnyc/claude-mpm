claude_mpm.utils.environment_context module

Environment context detection for smart dependency checking.

This module determines the execution environment and whether interactive prompting is appropriate for dependency installation.

class claude_mpm.utils.environment_context.EnvironmentContext[source]

Bases: object

Detects and analyzes the execution environment.

WHY: We need to know if we’re in an environment where prompting makes sense. Interactive prompting should only happen in TTY environments where a human is present. CI/CD, Docker, and non-interactive contexts should skip prompts.

DESIGN DECISION: Use multiple indicators to detect environment type: - TTY presence is the primary indicator for interactivity - Environment variables help identify CI/CD and containerized environments - Command-line flags can override automatic detection

CI_ENV_VARS = ['CI', 'CONTINUOUS_INTEGRATION', 'GITHUB_ACTIONS', 'GITLAB_CI', 'JENKINS', 'TRAVIS', 'CIRCLECI', 'BUILDKITE', 'DRONE', 'TEAMCITY_VERSION', 'TF_BUILD', 'CODEBUILD_BUILD_ID', 'BITBUCKET_BUILD_NUMBER']
CONTAINER_INDICATORS = ['/.dockerenv', '/run/.containerenv']
