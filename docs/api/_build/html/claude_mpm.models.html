Agent models package.

WHY: This package centralizes all data models used for agent management, providing a single source of truth for data structures across the system.

Bases: object

Complete agent definition.

WHY: This is the main model that represents an agent’s complete configuration: - Combines all aspects of an agent in one place - Provides a clear contract for what constitutes an agent - Makes serialization/deserialization straightforward

DESIGN DECISION: Using composition over inheritance: - AgentMetadata, AgentPermissions, and AgentWorkflow are separate classes - This allows each component to evolve independently - Services can work with just the parts they need
