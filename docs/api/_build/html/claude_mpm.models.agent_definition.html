claude_mpm.models.agent_definition module

Agent Definition Models

Data models for agent definitions used by AgentManager.

WHY: These models provide a structured representation of agent data to ensure consistency across the system. They separate the data structure from the business logic, following the separation of concerns principle.

DESIGN DECISION: Using dataclasses for models because: - They provide automatic __init__, __repr__, and other methods - Type hints ensure better IDE support and runtime validation - Easy to serialize/deserialize for persistence - Less boilerplate than traditional classes
