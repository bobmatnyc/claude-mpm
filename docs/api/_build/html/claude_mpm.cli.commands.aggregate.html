claude_mpm.cli.commands.aggregate module

CLI commands for the Event Aggregator service.

WHY: Provides command-line interface for managing the event aggregator service that captures Socket.IO events and saves them as structured session documents.

claude_mpm.cli.commands.aggregate.aggregate_command(args)[source]

Main entry point for aggregate commands.

WHY: Routes subcommands to appropriate handlers for managing the event aggregator service.

claude_mpm.cli.commands.aggregate.start_command(args)[source]

Start the event aggregator service.

WHY: Starts capturing events from the Socket.IO dashboard server for building complete session documents.

claude_mpm.cli.commands.aggregate.stop_command(args)[source]

Stop the event aggregator service.

WHY: Gracefully stops the aggregator and saves any active sessions.

claude_mpm.cli.commands.aggregate.status_command(args)[source]

Show status of the event aggregator service.

WHY: Provides visibility into what the aggregator is doing and what it has captured.

claude_mpm.cli.commands.aggregate.sessions_command(args)[source]

List captured sessions.

WHY: Shows what sessions have been captured for analysis.

claude_mpm.cli.commands.aggregate.view_command(args)[source]

View details of a specific session.

WHY: Allows detailed inspection of what happened during a session.

claude_mpm.cli.commands.aggregate.export_command(args)[source]

Export a session to a file.

WHY: Allows sessions to be exported for external analysis or sharing.

claude_mpm.cli.commands.aggregate.add_aggregate_parser(subparsers)[source]

Add the aggregate command parser.

WHY: Integrates the aggregator commands into the main CLI system.
