## v6.5.40 (2026-06-12)

### Behavior Changes

- **Bundled skills now install at the user level (`~/.claude/skills/`) instead of per-project.**
  A single `claude-mpm` startup deploys bundled skills (e.g. `session-analyzer`) once, making
  them available across all projects. Previously, a project that had never run `claude-mpm` would
  be missing bundled skills entirely. (#802, #804)

### Migration: `v6_5_40` (runs automatically on startup)

On startup, `claude-mpm` removes redundant project-level copies of bundled skills from the
**current project's** `.claude/skills/` directory, deferring to the user-level copy in
`~/.claude/skills/`. Each project is cleaned the next time `claude-mpm` starts up in that
project — cleanup is not applied to all projects at once.
A project-level skill is only removed when **both** conditions are true:

1. It is part of the bundled skill set that ships with `claude-mpm`, **and**
2. A user-level copy already exists in `~/.claude/skills/`.

Project-specific custom skills (anything you authored directly in a project's `.claude/skills/`)
are never touched.

**What you need to do:** Nothing. Deployment and cleanup happen automatically. After upgrading,
your bundled skills live in `~/.claude/skills/`; your own project-specific skills remain in
`<project>/.claude/skills/`.
