## Adding Troubleshooting Documentation

When investigating complex issues:

1. **Document the problem**: Clear problem statement
2. **Capture investigation steps**: What was checked and why
3. **Record findings**: Root cause analysis
4. **Document solution**: How it was resolved
5. **Add prevention**: How to avoid in future

## Template

```markdown
# [Issue Title]

## Problem Statement
Clear description of the issue

## Investigation
Steps taken to diagnose the problem

## Root Cause
What caused the issue

## Solution
How it was resolved

## Prevention
How to avoid this in the future
```
